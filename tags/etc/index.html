History

  • 브라우저 히스토리(세션 기록) 정보를 반환하거나 제어합니다.

  • 속성

    • .length : 등록된 히스토리 개수
    • .scrollRestoration : 스크롤 위치 복원 여부 - ‘auto’ / ‘manual’
      • auto : 스크롤 위치를 기억
      • manual : 스크롤 위치를 반영하지 않음.
    • .state : 현재 히스토리에 등록된 데이터(상태)

  • 메소드

    • 모든 브라우저(Safari 제외)는 .pushState()와 .replaceState()의 ‘제목’ 옵션을 무시. -> 추후에 활용될 가능성이 있으므로 빈 문자열을 전달하는 것을 권장.

    • .pushState()와 .replaceState() 호출은 popstate 이벤트가 발생하지 않음!!

    • .back() : 뒤로 가기

    • .forward() : 앞으로 가기

    • .go(위치) : 현재 페이지 기준 특정 히스토리 ‘위치’로 이동 - .go(-2)

    • .pushState(상태, 제목, 주소) : 히스토리에 상태 및 주소를 추가

      • 상태에 이동할 페이지에 전달할 데이터를 입력
      • 명시한 주소로 페이지를 이동.(=== location.assign(path) 동작)
      • 페이지를 이동한 후 state 매개변수에 전달한 데이터를 활용할 수 있다.(History.state)

    • .replaceState(상태, 제목, 주소) : 현재 히스토리의 상태 및 주소를 교체

      • 현재 위치한 페이지를 방문한 내역을 남기지 않고 이동할 때 활용. 언제..사용..?

  • 예제
    - 해시 링크를 활용한 페이지 내 이동 (서버 요청 X) 해시값과 동일한 id를 가진 요소로 스크롤 이동

이벤트 위임 (Delegation)

  • 비슷한 패턴의 여러 요소에서 이벤트를 핸들링해야 하는 경우, 단일 조상 요소에서 제어하는 이벤트 위임 패턴을 사용할 수 있음.
  • 대부분의 경우 이벤트 위임을 통한 성능 개선은 개수가 엄청 많을 때 !

커스텀 이벤트

  • 대상.dispatchEvent(이벤트)
  • 이벤트를 강제로 발생시킬 수 있음!!
  • 기본적으로 존재하는 이벤트 외에 사용자 정의 이벤트를 만들어 강제로 발생시키면 동작시킬 수 있다.
  • 이벤트를 발생시키면서 동시에 data를 전달할 수 있다.
  • 프레임워크에서 부모 컴포넌트와 자식 컴포넌트 사이에 data를 주고 받을 때 내부적으로 동작할 떄 사용된다.
  • 바닐라JS로 프로젝트를 진행할 때 컴포넌트끼리 data를 주고 받을 때 유용해보임.

Web APIs

Console

  • .log(), .warn(), .error(), .dir()

  • 콘솔에 메시지나 객체를 출력.

  • .count(), countReset()

  • 콘솔에 메소드 호출의 누적 횟수를 출력하거나 초기화.

  • .time(), .timeEnd()

  • 타이머가 시작해서 종료되기까지의 시간(ms)을 콘솔에 출력.

  • .trace()
  • 메소드 호출 스택(Call Stack)을 추적해 출력.

Cookie

  • 개요

    • 도메인 단위로 저장
    • 모든 웹 요청에 포함돼 서버로 전송(때문에 민감한 정보는 다루지 않는다)
    • 표준안 기준, 사이트당 최대 20개 및 4KB로 제한
    • 영구 저장 불가능

  • 옵션

    • domain: 유효 도메인 설정 - domain=localhost;
    • path: 유효 경로 설정 - path=/;
    • expires: 만료 날짜(UTC Date) 설정
    • max-age: 만료 타이머(s) 설정
    • secure: HTTPS 보안 프로토콜만 전송 - secure;
    • samesite: 크로스 사이트 요청 처리 여부(none, lax, strict) - samesite=none

  • 예제

  • 쿠키에 저장되어 있는 데이터를 get해오면 통 문자열로 넘어오기 때문에 문자열을 가공하는 함수를 이용하여 get한다.
  • 모달창 띄우는 동작을 제어할 때 사용해보기

Storage

  • 개요

    • 도메인 단위로 저장
    • 서버로 전송되지 않음
    • 5MB 제한
    • 세션 혹은 영구 저장 가능

  • 종류

    • sessionStorage : 브라우저 세션이 유지되는 동안에만 데이터 저장 (브라우저 끄면 날라감)
      • 브라우저 끌 때 날라가야만하는 구조일 때 사용하자.
    • localStorage : 따로 제거하지 않으면 영구적으로 데이터 저장(코드)
      • 로그인 세션에 활용
    • indexedDB
      • localStorage보다 용량이 크다는 장점이 필요할 때 사용하자.
      • 로컬에서 대규모 용량을 쓸 때.. (그럴 경우가 ..?)

  • 메소드

    • .getItem() : 데이터 조회
    • .setItem() : 데이터 추가
    • .removeItem() : 데이터 제거
    • .clear() : 스토리지 초기화

  • 예제

Location

  • 현재 페이지 정보를 반환하거나 제어.

  • 속성

    • .href: 전체 URL 주소
    • .protocol: 프로토콜 - ‘http:’ or ‘https:’
    • .hostname: 도메인 이름 - ‘google.com’
    • .pathname: 도메인 이후 경로 - ‘/a/b/c’
    • .host: 포트 번호를 포함한 도메인 이름 - ‘google.com:5500’
    • .port: 포트 번호 - ‘5500’
    • .hash: 해시 정보(페이지의 ID) - ‘’ or ‘#abc’

  • 메소드

    • 페이지가 항상 새로고침됩니다.
    • .assign(주소): 해당 ‘주소’로 페이지 이동
    • .replace(주소): 해당 ‘주소’로 페이지 이동, 현재 페이지 히스토리를 제거
    • .reload(강력): 페이지 새로고침, true 인수는 ‘강력’ 새로고침

Element.style

  • 요소의 style 속성(인라인 스타일)의 CSS 속성 값을 얻거나(get) 지정(set)

window.getComputedStyle()

  • 요소에 적용된(계산되어) 스타일 객체를 반환.
  • 바로 위의 Element.style은 인라인 스타일로 적용된 속성만 얻을 수 있는 문제를 해결.

크기와 좌표

예제 이해를 위한 기본 HTML/CSS 구조

window.innerWidth / window.innerHeight

  • 현재 화면(Viewport)의 크기를 Get !.

window.scrollX / window.scrollY

  • 페이지의 좌상단 기준, 현재 화면(Viewport)의 수평 혹은 수직 스크롤 위치를 Get !
  • 페이지 좌상단 기준, 스크롤을 얼마나 했니 ?
  • windows의 경우 디스플레이 해상도도 영향을 받을 수 있다.

window.scrollTo() / Element.scrollTo()

  • 지정된 좌표로 대상(화면, 스크롤 요소)을 스크롤.

Element.clientWidth / Element.clientHeight

  • 테두리 선(border)을 제외한 요소의 크기를 Get !
  • border 제외 === padding box ! (content-box + padding 값 !)
  • 혼동하지말 것 : padding-box라는 css 속성은 없다 (개념적으로 받아들이기)

Element.offsetWidth / Element.offsetHeight

  • 테두리 선(border)을 포함한 요소의 크기를 Get !
  • css의 border-box 와 동일 !

Element.scrollWidth / Element.scrollHeight

  • 요소의 테두리 선(border)을 제외한 스크롤 영역까지의 크기를 Get !

Element.scrollLeft / Element.scrollTop

  • 스크롤 요소의 좌상단 기준, 현재 스크롤 요소의 수평 혹은 수직 스크롤 위치를 Get !

Element.offsetLeft / Element.offsetTop

  • 페이지의 최좌상단 기준, 요소의 위치를 Get !
  • scrollTo 메서드와 함께 사용하면 해시 링크(페이지 내 링크)와 동일하게 동작하도록 구현할 수 있을 것 같다.

Element.getBoundingClientRect()

  • 테두리 선(border)을 포함한 요소의 크기와 화면에서의 상대 위치 정보를 Get !
  • 메서드가 호출되는 그 순간의 정보를 받아옴.

Data Format에 대해

Data Format (22.12.17)

Focus

  1. 데이터 포맷이 왜 필요할까 ?

  2. JSON

  3. YAML

  4. 데이터 직렬화/역직렬화

1. 데이터 포맷이 왜 필요할까 ?

  • 혼자 개발하거나, 내부적으로 데이터를 사용할 땐 내부의 규칙을 문서화하고 참고하면 그만이지만, 문제는 타인 또는 타 시스템과 연동을 해야할 때다. 데이터를 주고 받을 때 마다 각자 정해진 규칙을 문서로 주고 받는 것도 비효율적이고, 하나의 통일된 규칙이 없을 때 생기는 불필요한 문서들이 많아지기 마련임.
  • 이렇게 상호가 데이터를 효율적으로 다루고자 통일된 규칙을 가지는 ‘format’이 등장하게 된다.
  • XML, JSON, YAML 등이 있다.

2. JSON

  • JSON 데이터 포맷 예시
  • 데이터 직렬화에 유리하여 일반적으로 데이터 전송, AJAX 통신 등 API 통신에 주로 사용됨.
  • XML은 헤더와 태그 등으로 가독성이 떨어지고, 쓸데없이 용량을 잡아먹는다는 단점이 있었던 데 반해 JSON은 나은 가독성, 효율성을 가진다.
  • 주석을 지원하지 않는다.
  • 데이터 타입을 강제하려면 JSON 스키마로 보완해야한다.

3. YAML

  • YAML 데이터 포맷 예시
  • 주로 수정이 잦은 설정(conf) 파일에 사용됨.
  • XML, JSON 보다 직관적이라 가독성이 좋다.
  • 주석이 가능하다.
  • 데이터 타입을 명시해줄 수 있다.
  • 데이터 직렬화가 불편하다.

4. 데이터 직렬화/역직렬화 : (보완, 수정 요 - 운영체제)

  • 직렬화(Serialization)란, 참조 타입의 데이터를 디스크에 저장하거나, 네트워크 통신에 사용하기 위한 형태로 변환하는 작업임.
  • 역직렬화(Deserialization)란, 디스크에 저장된 데이터를 읽거나, 네트워크를 통해 전송받은 데이터를 메모리에 저장하기 위한 형태로 변환하는 작업임.
  • 메모리에서 데이터가 가지는 구조는 2가지다.
    • 값(Value) 형식 데이터
    • 참조(Reference) 형식 데이터 : 객체 타입이 여기에 해당.
  • 이 두가지 데이터 중 디스크에 저장하거나 네트워크 통신을 할 수 있는 데이터는 값(Value) 형식의 데이터다.
  • 직렬화를 통해 참조 형태의 데이터를 값 형태로 변환.
