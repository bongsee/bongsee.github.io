브라우저 렌더링 과정

  1. HTML 파싱
  2. DOM 트리 생성(DOMContentLoaded 이벤트)
  3. CSS 파싱
  4. CSSOM 트리 생성
  5. DOM-CSSOM 결합
  6. Render 트리 생성
  7. 레이아웃 계산
  8. Rendering(load 이벤트)

requestAnimationFrame(callback)

  • 브라우저의 리페인트 주기에 맞게 콜백을 실행
  • 브라우저마다, 실행환경마다 각각 리페인트(렌더링) 주기가 설정되어 있다.
  • 통상 10ms 내외라고 알려져 있음.

throttle / debounce (Lodash)

  • throttle : 일정 시간 간격으로 함수를 실행 (실행 횟수를 조절)
  • debounce : 일정 시간 동안 함수가 실행되지 않으면 함수를 실행
    • debounce 예시 ) 추천검색어 노출
    • 사용자가 입력중인데 모든 keydown 이벤트를 리스닝할 필요는 없는 것임을 이용. UX에 기반하여 적절하게 시간을 두어 직전 이벤트를 취소시키는 것

DOM

  • Node와 Element를 구분하는 것이 중요 Node라는 개념 안에 Element가 속해있음.

  • 왜 구분해야할까 ? 각각이 쓸 수 있는 프로토타입 메서드 혹은 정적 메서드가 다르기 때문임.

  • 실상 우리는 Element를 주로 다루기 때문에 메서드도 Element 관련 메서드를 자주 사용하게 될 것.

  • 새로 배운 메서드들

  • Element.closest('css selector')

    • 자신을 포함하여 지정한 CSS 선택자와 일치하는 가장 가까운 조상 요소를 반환. 요소를 찾지 못하면, null을 반환.

  • Element.children('css selector')

    • 요소의 모든 자식 요소를 반환.
    • 반환값 : HTMLCollection[]

  • document.createElement()

    • 메모리에만 존재하는 새로운 HTML 요소를 생성해 반환.

  • Element.remove()

    • 요소를 제거.

  • Element.dataset

    • 자바스크립트의 데이터를 html요소에 저장할 때 활용 (전역 속성) (게시글의 조회수 등등)
    • html의 dash-case => js에서 camelCase로 사용
    • json 데이터나 객체 데이터 등등을 html에 저장할 때는 html entity를 사용하자.(근데 사실상 js 코드에서 동적으로 부여하는 것이기에 html 파일에 직접 넣는 건 흔하지 않다고 함.)
    • 예시

  • .className(getter)

    • 여러개 있으면 공백하나로 구분된 문자열이 넘어오므로 사용할 거면 split해서 써야.
    • class를 관리하는 용도라면 .classList를 쓰는게 훨씬 편하겠다.

  • .classList(getter) –> getter,setter 익숙해지고자 굳이 쓰는..

    • .add()
    • .remove()
    • .toggle()
    • .contains()
